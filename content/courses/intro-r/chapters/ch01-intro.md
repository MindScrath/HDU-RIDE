# 第1章 导论

在现代金融学的学习中，我们往往面临着海量的数据。如何从这些数据中挖掘出有价值的规律，如何预测未来的市场走势，是每一个金融从业者和研究者必须思考的问题。金融计量学（Financial Econometrics）正是为了解决这些问题而诞生的一门交叉学科。本章将作为整本教材的起点，带领大家认识这门学科，掌握金融市场中最核心的变量——资产收益率的计算，并初步了解如何利用强大的开源软件 R 以及现代人工智能（AI）技术，开启我们的金融数据分析之旅。

## 1.1 课程介绍

### 1.1.1 学科发展

金融计量学是金融学、经济学、统计学和计算机科学的交叉学科。早在20世纪中叶，随着现代资产定价理论（如马科维茨的投资组合理论和资本资产定价模型 CAPM）的提出，学者们开始需要使用统计方法来检验这些理论在现实市场中是否成立。
随着计算机技术的飞速发展和金融市场数据的爆炸式增长（如高频交易数据），传统的统计方法逐渐显得力不从心。金融时间序列数据往往具有“波动率聚集”（Volatility Clustering）、“尖峰厚尾”（Fat Tails）等独特特征。为了专门处理这类数据，金融计量学逐渐脱离了传统的经济计量学，发展出一套以时间序列分析、波动率建模（如 ARCH/GARCH 模型）为核心的独立学科体系。

### 1.1.2 主要内容

本课程旨在通过理论与实践相结合的方式，系统性地讲授金融计量学的核心知识。考虑到很多同学可能是初次接触时间序列分析和编程，本书在编写时将秉持“深入浅出”的原则：
1. **理论推导详尽化**：对于复杂的数学模型，我们将拆解每一个推导步骤，确保大家“知其然更知其所以然”。
2. **代码实操场景化**：理论的最终目的是应用。我们将通过 R 语言，手把手教大家如何将公式转化为代码，如何在真实的金融市场上进行数据分析。
3. **前沿技术融合化**：本书特别引入了基于大语言模型（如 ChatGPT、Claude 等）的 AI 编程辅助教学。即使你是编程“小白”，也能在 AI 的帮助下快速实现复杂的数据处理。

本课程的主要内容包括：时间序列基础、ARMA 过程、一元与多元时间序列分析、时间序列回归模型，以及波动率模型等。

## 1.2 资产收益率

在金融计量学中，我们极少直接对资产的价格序列（如股票每天的收盘价）进行建模，而是对其“收益率”（Return）进行建模。为什么呢？一方面，投资者的核心诉求是“收益”；另一方面，价格序列通常是不平稳的（表现为随机游走，我们将在后续章节详细讨论），而收益率序列通常呈现出更好的统计特性（如平稳性），更适合进行计量分析。

在深入计算收益率之前，我们需要先了解资金的时间价值，即利率体系。

### 1.2.1 金融市场

金融市场是资金供给者和资金需求者进行交易的场所。无论是股票、债券、外汇还是衍生品，任何金融工具的交易都伴随着资金的跨期配置。在这个配置过程中，资金的使用是有成本的，这个成本就是**利率**（Interest Rate）。利率不仅是金融产品定价的基石，也是计算资产收益率时的重要参照。

### 1.2.2 利率体系

假设你在银行存入了一笔本金（Principal），记为 $P_0$。银行承诺给你的年利率为 $R$。我们来看看在不同计息方式下，这笔钱在 $n$ 年后的终值（Future Value, 记为 $FV_n$）是多少。

**(1) 单利 (Simple Interest)**

单利是指只对本金计算利息，之前产生的利息不再产生新的利息。
第一年年末，你获得的利息为 $P_0 \times R$，账户总额为：
$$FV_1 = P_0 + P_0 \times R = P_0(1 + R)$$
第二年年末，利息依然是 $P_0 \times R$，账户总额为：
$$FV_2 = P_0 + P_0 \times R + P_0 \times R = P_0(1 + 2R)$$
依此类推，$n$ 年后的终值为：
$$FV_n = P_0(1 + nR)$$

**(2) 复利 (Compound Interest)**

现实金融市场中，更常见的是复利，即“利滚利”。之前产生的利息会加入本金中，共同产生下一期的利息。
第一年年末，账户总额为：
$$FV_1 = P_0(1 + R)$$
第二年年末，是以第一年年末的总额 $FV_1$ 作为新的本金来计算利息，因此：
$$
\begin{aligned}
FV_2 &= FV_1(1 + R) \\&= P_0(1 + R)(1 + R) \\&= P_0(1 + R)^2
\end{aligned}
$$
依此类推，$n$ 年后的终值为：
$$FV_n = P_0(1 + R)^n$$

**(3) 计息频次与连续复利 (Continuous Compounding)**

在上面的复利计算中，我们默认是“一年计息一次”。但如果银行说，我们每半年计息一次，但给你的年化报价利率依然是 $R$ 呢？
这意味着，每半年的利率是 $R/2$，而一年内会计息 2 次。因此，一年后的终值为：
$$FV_1 = P_0(1 + \frac{R}{2})^2$$
如果每个月计息一次（一年计息 12 次），那么一年的终值为：
$$FV_1 = P_0(1 + \frac{R}{12})^{12}$$
一般地，假设一年内计息 $m$ 次，那么 $n$ 年后（共计息 $m \times n$ 次），终值的计算公式为：
$$FV_n = P_0(1 + \frac{R}{m})^{m \times n}$$

现在，让我们进行一个极限思考：如果计息的频次无限大，即 $m \rightarrow \infty$，也就是每一分、每一秒、甚至每一个无穷小的瞬间都在计息，终值会变成多少呢？
我们将上述公式稍作变形，令 $k = \frac{m}{R}$，则当 $m \rightarrow \infty$ 时，$k \rightarrow \infty$：
$$
\begin{aligned}
FV_n &= \lim_{m \to \infty} P_0(1 + \frac{R}{m})^{m \times n} \\&= P_0 \lim_{k \to \infty} \left[ (1 + \frac{1}{k})^k \right]^{R \times n}
\end{aligned}
$$
根据微积分中的重要极限，我们知道自然底数 $e$ 的定义为：
$$\lim_{k \to \infty} (1 + \frac{1}{k})^k = e \approx 2.71828$$
将自然底数代入，我们得到了金融学中极其重要的**连续复利终值公式**：
$$FV_n = P_0 e^{R \times n}$$
连续复利在理论金融学（尤其是期权定价和金融工程）中应用极为广泛，因为它在数学处理上（如求导、积分）比离散的复利要方便得多。

### 1.2.3 市场模型与收益率计算

假设某只股票在时刻 $t-1$ 的价格为 $P_{t-1}$，在时刻 $t$ 的价格为 $P_t$。在 $t-1$ 到 $t$ 期间，该股票派发了红利 $D_t$。

**(1) 简单收益率 (Simple Return)**

从 $t-1$ 到 $t$ 这一期的简单净收益率（记为 $R_t$）定义为资产价值的变化量除以初始投资额：
$$R_t = \frac{(P_t - P_{t-1}) + D_t}{P_{t-1}} = \frac{P_t + D_t}{P_{t-1}} - 1$$
如果我们不考虑红利（或者将红利再投资已包含在价格除权中），公式简化为：
$$R_t = \frac{P_t - P_{t-1}}{P_{t-1}} = \frac{P_t}{P_{t-1}} - 1$$
为了方便计算，我们常常定义**总收益率**（Gross Return），记为 $1 + R_t$：
$$1 + R_t = \frac{P_t}{P_{t-1}}$$

简单收益率非常符合直觉，但它在多期计算时存在一个明显的缺点：不具备时间可加性。
假设有两期收益，第一期总收益为 $1+R_1 = \frac{P_1}{P_0}$，第二期总收益为 $1+R_2 = \frac{P_2}{P_1}$。那么两期的累积总收益率为：
$$
\begin{aligned}
1 + R_c &= \frac{P_2}{P_0} \\&= \frac{P_1}{P_0} \times \frac{P_2}{P_1} \\&= (1+R_1)(1+R_2) \\&= 1 + R_1 + R_2 + R_1R_2
\end{aligned}
$$
可以看出，两期的累积净收益率 $R_c \neq R_1 + R_2$，它是一个乘积的形式，这在处理长期数据时会带来很多数学上的不便。

**(2) 对数收益率 (Log Return 或 Continuously Compounded Return)**

为了解决简单收益率不具备时间可加性的问题，我们引入**对数收益率**，通常记为 $r_t$。定义为总收益率的自然对数：
$$r_t = \ln(1 + R_t) = \ln(\frac{P_t}{P_{t-1}}) = \ln(P_t) - \ln(P_{t-1})$$
如果我们用小写字母 $p_t$ 表示对数价格，即 $p_t = \ln(P_t)$，那么对数收益率可以极其优雅地表示为对数价格的一阶差分：
$$r_t = p_t - p_{t-1}$$

对数收益率完美地解决了时间可加性的问题。考察两期的累积对数收益率：
$$
\begin{aligned}
r_c &= \ln(\frac{P_2}{P_0}) \\&= \ln(\frac{P_1}{P_0} \times \frac{P_2}{P_1}) \\&= \ln(\frac{P_1}{P_0}) + \ln(\frac{P_2}{P_1}) \\&= r_1 + r_2
\end{aligned}
$$
推广到 $k$ 期，多期的对数收益率等于各期对数收益率之和：
$$r_c = r_1 + r_2 + \cdots + r_k$$
这一“可加性”的优美性质，使得对数收益率成为了金融计量学研究中的“标准配置”。本书后续章节中所提到的“收益率”，除非特别说明，均指代**对数收益率**。

### 1.3.2 基于R的金融数据处理

**常用的R包**
R 的强大在于其庞大的扩展包（Packages）。在本书的后续学习中，我们会频繁使用到以下几个核心包：

- `xts` 和 `zoo`：经典的时间序列数据结构处理包。

- `pedquant`：一个非常优秀的量化金融包，特别适合用来获取中国 A 股及全球市场的金融数据。

在使用包之前，我们需要先安装它（一台电脑只需安装一次），然后加载它（每次打开 RStudio 都需要加载）：
```R
# 安装包（只需要运行一次，如果已经安装过则不需要取消注释运行）
# install.packages("tidyverse")
# install.packages("pedquant")

# 加载包
library(tidyverse)
library(pedquant)
```

**Pedquant 包介绍与 A 股数据拉取**
获取高质量的金融数据往往是数据分析的第一步。过去我们可能需要去雅虎金融（Yahoo Finance）下载数据，但受限于网络环境，国内用户体验并不稳定，且时常缺乏 A 股的完整数据。`pedquant` 包完美解决了这个问题，它可以方便地从国内财经接口无缝抓取最新的 A 股数据。

让我们来实际拉取一下“贵州茅台”（股票代码：600519）自 2020 年初至 2023 年末的日线数据。

```R
# 使用 pedquant 包的 md_stock 函数获取贵州茅台的数据
# 600519 后面加上 .SH 表示上海证券交易所 (Shanghai)
moutai_data <- md_stock(symbol = "600519.SH", 
                        from = "2020-01-01", 
                        to = "2023-12-31")

# 提取返回列表中的第一个元素（即数据框本身）
moutai_df <- moutai_data[[1]]

# 查看前几行数据，了解数据结构
head(moutai_df)
```

数据拉取下来后，我们会看到开盘价(open)、最高价(high)、最低价(low)、收盘价(close)等字段。接下来，我们计算我们在前一节学过的**对数收益率**。在这里，我们将介绍 R 中非常著名的“管道符” `%>%`（或新版的 `|>`）。你可以把管道符理解为工厂里的传送带，将上一步处理好的数据直接传递给下一步进行加工，这让代码读起来就像人类语言一样顺畅。

```R
# 使用 dplyr 清洗数据并计算对数收益率
moutai_clean <- moutai_df %>%
  # 第一步：按日期排序，确保时间序列的先后顺序完全正确
  arrange(date) %>%
  # 第二步：增加新的列（mutate），计算对数收益率
  # log(close / lag(close)) 即为当天收盘价除以前一天收盘价，再取自然对数
  mutate(
    log_return = log(close / lag(close))
  ) %>%
  # 第三步：由于第一天没有前一天的数据，会产生 NA（缺失值），我们将包含 NA 的行删除
  drop_na()

# 查看计算后的核心列
head(moutai_clean %>% select(date, close, log_return))
```

**数据可视化**
金融数据分析绝对离不开画图。我们使用 R 语言中最强大的画图工具 `ggplot2`，为贵州茅台的收盘价和对数收益率分别绘制折线图，并将它们组合在一起，以便于直观观察。

```R
library(cowplot) # 这是一个用于组合多张图的优秀包

# 1. 绘制收盘价折线图
p_price <- moutai_clean %>%
  ggplot(aes(x = date, y = close)) +
  geom_line(color = "darkred", linewidth = 0.8) + # 设置线条颜色和粗细
  labs(title = "贵州茅台 (600519.SH) 历史收盘价", x = "日期", y = "价格 (元)") +
  theme_bw() # 使用简洁的黑白主题，看起来更学术

# 2. 绘制对数收益率折线图
p_return <- moutai_clean %>%
  ggplot(aes(x = date, y = log_return)) +
  geom_line(color = "steelblue", linewidth = 0.5) +
  labs(title = "贵州茅台对数收益率", x = "日期", y = "对数收益率") +
  theme_bw()

# 3. 使用 cowplot 将两张图上下拼接
plot_grid(p_price, p_return, nrow = 2, align = 'v')
```
*注：您可以将上述代码复制到您的 RStudio 中运行，体会代码生成直观图表的魅力。观察下方的收益率图，您可以初步体会到何为**“波动率聚集”**（Volatility Clustering）——即大的波动往往跟随着大的波动，小的波动往往跟随着小的波动，这是金融时间序列极其典型的特征之一。*

### 1.4.2 基于AI的数据清洗与异常处理

在处理实际金融数据时，真实世界的数据往往是“脏”的。比如，我们经常会遇到停牌导致的数据缺失（NA 值），或者多只不同股票由于停牌日不同导致交易日期不对齐的问题。遇到这种“脏活累活”，直接向 AI 求助是最高效的：

> **💡 小白的 AI Prompt 示例 2：数据清洗**
> “我有一个 R 语言的 dataframe 叫 `stock_data`，里面有 `date`, `close` 两列数据。其中 `close` 列中间有几天是 NA（因为股票停牌了没有价格）。按照金融分析的惯例，我想用停牌前一天的收盘价来向下填补这些 NA 值。在 R 的 `tidyverse` 体系下应该用什么函数最方便？请给我一段简短的示例代码。”

AI 马上会告诉你使用 `tidyr` 包中的核心函数 `fill(close, .direction = "down")`，并给你提供测试代码。这就免去了你在网上苦苦搜索的时间。

### 1.4.3 基于AI的报错排查

“Debug”（排除代码错误）是写代码最耗时的环节。小白最害怕 RStudio 控制台里突然弹出的红字报错。在 AI 时代，最佳实践是：**不要慌张，直接把红字报错信息完整复制，丢给 AI 解释。**

> **💡 小白的 AI Prompt 示例 3：错误排查**
> “我在运行 R 代码计算收益率时，系统报了如下错误：
> `Error in mutate() : could not find function "mutate"`
> 请问这是什么原因？我该如何解决这个问题？”

AI 会像一位耐心的助教一样，精准地指出：“这是一个非常常见的错误。系统找不到 `mutate` 函数，通常是因为你**忘记加载相应的包**了。`mutate` 是 `dplyr` 包里的函数，请确保在代码的最前面运行了 `library(dplyr)` 或者 `library(tidyverse)`。”
