# 需求文档（可落地版）

## 1. 项目背景与目标

该项目用于杭电计量金融课程教学。课程以 R 语言教学为主，目标是提供一个基于 Web 的教学平台，将 RStudio Server 与教学管理（讲义、作业、批改、班级与成员管理）整合到同一页面中，降低上手成本并提升教学效率。

技术栈约束：
- 前端：Vue + Element Plus
- 后端：Go + Gin
- 数据库：PostgreSQL
- 缓存：一期不强制，引入 Redis 作为可选优化

页面布局目标：
- 左侧：课程侧边栏（讲义、作业、班级、管理等）
- 右侧：RStudio 交互区（iframe/反向代理集成）

## 2. 范围（Scope）

### 2.1 一期必须实现（MVP）
- 用户登录、退出、会话保持
- 四类角色 + root 账号（仅开发/运维使用）
- 班级（创建、成员导入/移除、助教指定）
- 讲义：Markdown 上传、按 Chapter/Section 组织、渲染与访问控制
- 作业：Markdown 上传、发布/截止时间、学生提交（文件/文本）、提交记录
- 批改：教师/助教查看提交内容、评分、评语、发布成绩
- 管理员：用户增删改查、角色调整
- RStudio 集成：平台页面可打开 RStudio（至少做到“同站点可访问 + 基本鉴权”）

### 2.2 二期建议实现
- 原子化权限/权限点配置
- 作业查重、自动评测（R 脚本单测）
- 细粒度审计日志
- Redis 缓存、消息队列（异步渲染/压缩/评测）
- 课程/学期维度、多课程支持

### 2.3 明确不做（一期不承诺）
- 公网开放注册
- SSO 自动登录到 RStudio（若无 RStudio Pro/特殊认证能力则不保证）
- 直接接管学生“实时在线会话”的远程协作（一期以“提交快照 + 可复现环境”替代）

## 3. 术语与对象

- 用户（User）：平台登录主体，仅拥有一个角色
- 角色（Role）：root / Admin / Teacher / Assistant / Student
- 班级（Class）：教师创建的教学单元，学生与助教隶属于班级
- 讲义（Handout）：按 Chapter/Section 组织的 Markdown 内容
- 作业（Assignment）：教师发布的任务，描述为 Markdown
- 提交（Submission）：学生对某次作业的提交快照（支持多次提交）
- 评分（Grade）：对某次提交的分数与评语

## 4. 角色与权限（可执行矩阵）

约束：
- 每个用户只能拥有一个角色
- 权限等级：root > Admin > Teacher > Assistant > Student
- Assistant 权限范围受“班级绑定”限制（只能管理被绑定班级）

核心能力矩阵（✅允许 / ❌禁止 / 🔒仅限所属班级）：

| 功能 | root | Admin | Teacher | Assistant | Student |
|---|---|---|---|---|---|
| 登录/退出 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 用户管理（增删改查/改角色） | ✅ | ✅ | ❌ | ❌ | ❌ |
| 创建班级 | ✅ | ✅ | ✅ | ❌ | ❌ |
| 班级成员管理（导入/移除学生） | ✅ | ✅ | ✅ | 🔒 | ❌ |
| 指定助教 | ✅ | ✅ | ✅ | ❌ | ❌ |
| 发布讲义（Chapter/Section） | ✅ | ✅ | ✅ | 🔒（可选） | ❌ |
| 查看讲义 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 发布作业 | ✅ | ✅ | ✅ | 🔒（可选） | ❌ |
| 查看作业 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 提交作业 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 查看提交情况（全班） | ✅ | ✅ | ✅ | 🔒 | ❌ |
| 批改/评分 | ✅ | ✅ | ✅ | 🔒 | ❌ |
| 查看成绩/评语 | ✅ | ✅ | ✅ | ✅ | ✅（仅自己） |

说明：
- “Assistant 是否可发布讲义/作业”：建议一期默认不开放，或由教师在班级中配置开关。

## 5. 业务流程（落地描述）

### 5.1 登录与落地页
- 输入账号/密码登录
- 登录成功后根据角色跳转：
  - root/admin：管理控制台（用户管理、系统状态）
  - teacher/assistant：教师端（班级、讲义、作业、批改）
  - student：学生端（讲义、作业、提交、成绩）

### 5.2 班级与成员
- Teacher 创建班级：名称、学期/备注（可选）
- 成员导入：支持 CSV 导入（学号/姓名/账号/初始密码），导入即创建 Student 用户并绑定班级
- 指定助教：从已有用户中选择 Assistant 并绑定到班级（或将某用户角色改为 Assistant 并绑定）

### 5.3 讲义（Chapter/Section）
- Teacher 创建 Chapter：标题、排序号（可选）
- Teacher 在 Chapter 下创建 Section：
  - 元信息：标题、描述、排序号
  - 内容：上传 Markdown 文件（.md）
- Student/Assistant/Teacher 查看：服务端将 Markdown 渲染为 HTML 返回前端展示

文件规则（建议一期强制）：
- 仅允许 .md
- 单文件大小限制（例如 2–5MB）
- 图片资源：一期可只支持外链；如需本地图片，按“资源上传 + URL 引用”方式单独实现

### 5.4 作业与提交
- Teacher 发布作业：
  - 标题、描述（Markdown）、发布时间、截止时间、是否允许多次提交
  - 可选：附件（.md / .zip）
- Student 提交作业：
  - 提交内容：支持（1）上传文件（如 .R/.Rmd/.zip）或（2）文本粘贴（Markdown/代码）
  - 每次提交生成一条 Submission 记录，保留历史
- 截止策略：
  - 截止后默认不允许提交；可由教师手动允许补交（记录补交标记）

### 5.5 批改与复现（替代“实时学生会话”）

一期可实现的“可复现批改”定义：
- 教师/助教在作业详情中看到班级学生列表与最新提交状态
- 点击某学生进入“提交详情”：
  - 左侧/上方：提交内容预览（文件列表、文本、提交时间、提交次数）
  - 右侧：打开 RStudio（教师自己的会话），并提供“一键下载/挂载提交内容”的方式，方便在 RStudio 中运行与修改
- 打分：
  - 分数（0–100 或自定义满分）
  - 评语（文本）
  - 可选：附件反馈（批注后的文件/截图）

备注：
- “教师直接看到学生当前 RStudio 工作环境并可直接修改”：若使用开源 RStudio Server，通常无法安全、稳定地无缝接管学生实时会话。一期采用“提交快照 + 可复现环境”作为可落地替代；二期再评估更深度的会话级集成方案。

### 5.6 学生查看批改结果
- Student 在作业中查看：
  - 最近一次提交
  - 分数、评语
  - 历史提交与对应评分（如允许多次）

## 6. RStudio Server 集成策略（实现约束写清楚）

一期目标：在平台右侧可访问 RStudio，且权限与平台用户体系尽量一致。

推荐实现路径（按可行性排序）：
1) 统一账号但不做 SSO：平台账号与 RStudio（Linux/PAM）账号同名同密码，用户在 iframe 内手动登录一次
2) 反向代理隔离：平台通过 Nginx/Traefik 将 /rstudio/ 代理到 RStudio，统一域名，减少跨域与 Cookie 问题
3) 更深度 SSO：需要评估 RStudio 的认证方案与部署形态（可能依赖 Pro 能力或额外组件），一期不作为承诺

对“批改时访问学生环境”的一期口径：
- 以 Submission 文件快照作为批改输入；教师在自己的 RStudio 会话中打开/运行该快照

## 7. 数据与存储（建议一期开箱即用）

### 7.1 数据库实体（建议）
- users：账号、密码哈希、角色、状态、创建时间
- classes：班级信息、创建者
- class_members：班级-用户关系（学生/助教）、加入时间
- chapters / sections：讲义组织结构与内容文件引用
- assignments：作业元信息、截止策略
- submissions：作业提交记录（指向文件存储或文本内容）
- grades：分数、评语、批改人、发布时间

### 7.2 文件存储（建议）
- 本地磁盘存储（一期）：按环境变量配置根目录，例如 data/
- 目录建议：
  - handouts/{classId}/{chapterId}/{sectionId}.md
  - assignments/{classId}/{assignmentId}/statement.md
  - submissions/{classId}/{assignmentId}/{userId}/{submissionId}/...

## 8. 安全与非功能性需求（必须写清楚）

- 密码安全：仅存储强哈希（bcrypt/argon2），禁止明文
- root/admin 测试账号：仅用于开发/测试环境，通过初始化脚本/配置开关启用；生产必须禁用默认弱密码
- 权限校验：后端接口必须进行角色校验 + 班级归属校验（Assistant/Student）
- 上传安全：文件类型白名单、大小限制、路径穿越防护、病毒扫描（二期可选）
- 日志：记录登录、导入、发布、评分等关键操作（至少落库或文件）
- 性能目标（一期建议）：100 并发以内可用；大文件上传限制可控

## 9. 页面与导航（建议）

- 通用：登录页、个人信息/退出
- Student：
  - 讲义列表 → 章节 → Section 阅读
  - 作业列表 → 作业详情 → 提交 → 查看成绩
- Teacher/Assistant：
  - 班级列表 → 班级详情（成员/助教）
  - 讲义管理（Chapter/Section）
  - 作业管理（发布/查看/导出提交）
  - 批改（按作业/按学生）
- Admin/root：
  - 用户管理（搜索、创建、重置密码、禁用）
  - 系统配置（可选）

## 10. 验收标准（可测试）

- 权限：不同角色访问同一资源时，符合“权限矩阵”与“班级归属”约束
- 讲义：教师上传后学生可渲染查看；Markdown 渲染不执行危险脚本
- 作业：发布、截止、提交策略生效；提交历史可追溯
- 批改：教师对提交评分后学生可查看；评分修改有记录（至少更新时间）
- 管理：管理员可创建/禁用用户、调整角色；导入学生可直接登录
- RStudio：平台页面可打开 RStudio；同域访问，无明显跨域问题
